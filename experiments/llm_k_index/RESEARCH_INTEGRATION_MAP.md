## 🧭 Research Navigation: From Question to Answer

This map shows how to use the integrated framework to answer research questions:

### Question Type 1: "Does Model X have Property Y?"

**Path**: Static K-Index → Dynamic → Multi-Scale
1. **Compute 8D K-Index** (static baseline)
2. **Track trajectories** (is property stable?)
3. **Analyze scales** (is property scale-invariant?)

**Example**: "Does GPT-5.1 have coherent reasoning?"
- K_H = 0.92 ✓ (static)
- K_H coherence = 0.85 ✓ (stable over time)
- H1 loops at all scales ✓ (scale-invariant)
- **Answer**: Yes, with high confidence

### Question Type 2: "What causes Property Y?"

**Path**: Contrastive → Causal → Mechanistic
1. **Compare models** with/without property
2. **Identify factors** correlated with property
3. **Run interventions** to test causation
4. **Mechanistic explanation** of how factors work

**Example**: "What causes high K_A (agency)?"
- Compare models by K_A → RLHF models score higher
- Hypothesis: RLHF increases agency
- Intervention: Test base vs RLHF versions → confirmed
- Mechanism: RLHF reward function encourages initiative
- **Answer**: RLHF causally increases K_A by +0.18 ± 0.04

### Question Type 3: "Can Property Y predict Capability Z?"

**Path**: Predictive → Validation → Refinement
1. **Train predictor** Y → Z
2. **Validate** on held-out models
3. **Refine** with dynamic/multi-scale features
4. **Mechanistic explanation** of why it works

**Example**: "Does K_Topo predict coding ability?"
- Train: K_Topo → HumanEval (R² = 0.75)
- Validate: Held-out models within ±3%
- Refine: Add coherence + scale features (R² = 0.86)
- Mechanism: Loops = reasoning cycles = algorithmic thinking
- **Answer**: Yes, R² = 0.86 with mechanistic understanding

### Question Type 4: "How does Model X differ from Model Y?"

**Path**: Dynamic + Multi-Scale + Contrastive
1. **Compare trajectories** (temporal differences)
2. **Compare scale signatures** (structural differences)
3. **Contrastive analysis** (causal factors)
4. **Synthesis** (comprehensive profile)

**Example**: "GPT-5.1 vs Claude Opus 4.5?"
- Dynamic: GPT more stable, Claude more adaptive
- Multi-Scale: GPT word-level weak, Claude strong all scales
- Contrastive: Architecture differences (attention patterns)
- **Answer**: GPT optimized for consistency, Claude for flexibility

### Question Type 5: "How can we improve Property Y?"

**Path**: Mechanistic → Causal → Validation
1. **Identify mechanisms** creating property
2. **Design intervention** based on mechanism
3. **Causal test** of intervention
4. **Validate** on multiple models

**Example**: "How to increase K_I (integration)?"
- Mechanism: K_I correlates with training diversity
- Intervention: Train on more diverse corpus
- Causal test: Diverse→ K_I = +0.21 (p < 0.01)
- Validation: Works across architectures
- **Answer**: Increase training diversity by 2-3x
